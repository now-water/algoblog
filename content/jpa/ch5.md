# 방향 : 단방향, 양방향

- 방향은 객체관계에만 존재하고, 테이블 관계는 항상 양방향이다.

- 회원 -> 팀 or 팀 -> 회원

* 회원 <-> 팀, (객체에선 서로 다른 단방향이 2개 - 양쪽에서 서로 참조)

# 다중성

- 다대일(N:1), 일대다(1:N), 일대일(1:1), 다대다(N:N)

# 연관관계의 주인

- 객체를 양방향 연관관계로 만들면 연관관계의 주인을 정해야한다.
